# Bidra til Wils Linux Legacy

Vi ønsker alle bidrag velkommen som hjelper til med å gjøre eldre maskinvare mer effektiv!

## Slik bidrar du:
1. Fork repositoriet.
2. Opprett en ny branch for din optimalisering (`git checkout -b optimalisering-navn`).
3. **Viktig:** Test endringene dine grundig på faktisk maskinvare før du sender en Pull Request. Reelle målinger (f.eks. boot-tider eller ressursbruk) må dokumenteres.
4. Send en Pull Request!

# Contributing to Wils Linux Legacy

We welcome all contributions that help make legacy hardware more efficient, stable, and streamlined using "The Edison Way"!

## Guidelines for the Path We Follow:

1. **Path of Least Resistance:** All scripts and configurations must be optimized for the lowest possible resource usage. Avoid unnecessary background noise (bloat).
2. **Directory Structure:** Before proposing new features or scripts, ensure they fit into the established folder architecture (`scripts/`, `configs/`, or `docs/`).
3. **Real-World Testing:** Changes must be tested on actual hardware. Please document performance improvements or stability gains in your Pull Request.

Thank you for helping to breathe new life into older hardware with open-source freedom!
